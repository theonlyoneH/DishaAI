## рЯУМ Project Overview

**Disha AI** is a semantic segmentation pipeline built for off-road autonomous Unmanned Ground Vehicle (UGV) navigation. Trained entirely on **synthetic desert imagery** generated by Duality AI's Falcon platform, the system segments 10 desert terrain classes and generalizes to unseen real-world desert environments.

### рЯОѓ Objective

Identify and segment the following classes from off-road desert scenes:

| Class | Class | Class |
|---|---|---|

## рЯПЧпЄП Model Architecture Progression

We benchmarked **three distinct Transformer-based architectures** across development sprints:

---

### Model 1 вАФ DINOv2-Small + ConvNeXt Head *(Baseline)*
> **Notebook:** `Model1_TRAIN_TEST.ipynb`

- **Backbone:** `dinov2_vits14` (~21M parameters)
- **Head:** Custom ConvNeXt-style segmentation head
- **Purpose:** Rapid proof-of-concept; establishes the data pipeline
- **Strengths:** Fast training, efficient memory use, learns macro features (Sky, Grass)
- **Limitations:** Blurry boundaries for small objects (Logs, Flowers) due to 1/14 spatial resolution

**Results:**
- Val IoU: ~0.19 | Dice: ~0.26 | Pixel Accuracy: ~0.61

---

### Model 2 вАФ DINOv2-Large + DeepLabV3+ ASPP *(Heavy Context)*
> **Notebook:** `Model2_TRAIN_TEST.ipynb`

- **Backbone:** `dinov2_vitl14_reg` (~307M parameters)
- **Head:** DeepLabV3+ with Atrous Spatial Pyramid Pooling (ASPP)
  - 5 dilation rates: `[1, 6, 12, 18, 24]`
  - Squeeze-and-Excitation (SE) channel attention module
- **Training Tricks:**
  - Gradient Accumulation: `8 steps √Ч batch 8 = effective batch 64`
  - Last 6 ViT blocks unfrozen for domain adaptation
  - AdamW + Cosine Annealing with Warm Restarts (SGDR)

**Results:**
- Val IoU: ~0.42 | Dice: ~0.60 | Pixel Accuracy: ~0.79
- Strong on Sky (0.980) and Landscape (0.540); weak on Logs/Flowers

---

### Model 3 вАФ SegFormer MiT-B2 *(Final Best Model)*
> **Notebook:** `Model_3_Train_and_Test.ipynb`

- **Backbone:** `nvidia/segformer-b2-finetuned-ade512-512`
- **Architecture:** Hierarchical Mix Transformer (MiT-B2) + lightweight All-MLP decoder
- **No positional encoding** вАФ naturally robust to resolution changes

**Fine-Tuning Techniques:**
- Differential Learning Rates: encoder @ `lr √Ч 0.1`, decoder @ full `lr`
- First 2 patch embedding blocks frozen (prevents catastrophic forgetting)
- OneCycleLR scheduler: warmup вЖТ peak `6e-4` вЖТ decay

**Loss Function вАФ FocalDiceLoss:**
```
Loss = 1.0 √Ч Focal(ќ≥=2.0) + 0.8 √Ч Dice
```
- Focal Loss: penalizes easy examples (Sky), focuses on hard classes (Logs, Flowers)
- Dice Loss: directly optimizes IoU for crisp boundaries

**Results (25 epochs):**
- **Val mIoU: 0.424** вЬЕ
- Pixel Accuracy: ~0.79
- Best per-class: Sky (0.926), Dry Bushes (0.375), Dry Grass (0.369)

## рЯФђ Data Augmentation Pipeline (`JointAugment`)

A joint spatial + photometric augmentation pipeline to close the **sim-to-real gap**:

```python
# Spatial
transforms.RandomHorizontalFlip()
transforms.RandomVerticalFlip()
transforms.RandomResizedCrop(size=512)

# Photometric
transforms.ColorJitter(brightness=0.4, contrast=0.4)

# Occlusion Simulation
transforms.RandomErasing(p=0.5)
```

> вЪ†пЄП **Interpolation Rule:** Images use `BILINEAR`; masks strictly use `NEAREST` to prevent boundary class corruption.

## вЪЩпЄП Hardware-Aware Training

| Constraint | Solution |
|---|---|
| 14GB VRAM (Colab T4 / Kaggle) | Gradient Accumulation (8 steps) |
| OOM at 512√Ч512 | AMP (Automatic Mixed Precision) |
| Exploding gradients in ViTs | Gradient Clipping (`max_norm=1.0`) |
| Class imbalance (Sky dominates) | FocalDiceLoss + manual class weights |
| Rare class blindness (Logs, Flowers) | Upweighted: Logs√Ч3.0, Flowers√Ч2.5 |

## рЯРЫ Key Engineering Challenges

### 1. Mask Value Encoding
Raw mask pixel values (`100, 200, ..., 10000`) caused PyTorch CrossEntropy index crashes.
```python
VALUE_MAP = {100: 0, 200: 1, 300: 2, ...}  # Remapped to 0вАУ10
```

### 2. Class Imbalance
Sky and Landscape dominated predictions. Solved with dual FocalDiceLoss + class weight arrays.

### 3. Transformer Gradient Instability
ViTs are brittle during fine-tuning. Solved with gradient clipping + OneCycleLR warm-up (30% warmup phase).
